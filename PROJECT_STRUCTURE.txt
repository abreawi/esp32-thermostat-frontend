API ENDPOINTS
=============

Autenticación (/api/auth)
├── POST   /register              # Registrar nuevo usuario
├── POST   /login                 # Iniciar sesión
└── GET    /me                    # Obtener usuario actual (protegido)

Dispositivos (/api/devices) [TODOS PROTEGIDOS]
├── POST   /claim                 # Reclamar dispositivo por MAC
├── GET    /                      # Listar dispositivos del usuario
├── GET    /:id                   # Obtener dispositivo específico
├── PUT    /:id                   # Actualizar alias del dispositivo
└── DELETE /:id/unclaim           # Desvincular dispositivo

MQTT (/api/mqtt) [PROTEGIDO]
└── POST   /publish               # Publicar mensaje MQTT

WebSocket
└── ws://localhost:3000?token=xxx # Updates en tiempo real


MQTT TOPICS (por dispositivo)
==============================

Comandos (Web → ESP32)
├── ESP32_AABBCCDDEEFF/command/temp      # Temperatura objetivo
├── ESP32_AABBCCDDEEFF/command/mode      # Modo (manual/programado)
├── ESP32_AABBCCDDEEFF/command/schedule  # Programaciones
└── ESP32_AABBCCDDEEFF/command/relay     # Control relay

Estado (ESP32 → Web)
├── ESP32_AABBCCDDEEFF/status/temp_current  # Temperatura actual
├── ESP32_AABBCCDDEEFF/status/temp_target   # Temperatura objetivo
├── ESP32_AABBCCDDEEFF/status/mode          # Modo actual
├── ESP32_AABBCCDDEEFF/status/relay         # Estado relay
├── ESP32_AABBCCDDEEFF/status/config        # Config completa (JSON)
└── ESP32_AABBCCDDEEFF/online               # Estado online/offline


BASE DE DATOS (SQLite)
======================

users
├── id (PRIMARY KEY)
├── email (UNIQUE)
├── password_hash (bcrypt)
├── name
└── created_at

devices
├── id (PRIMARY KEY)
├── mac_address (UNIQUE)
├── topic_prefix (ESP32_AABBCCDDEEFF)
├── device_name
├── is_online
├── last_seen
└── created_at

user_devices (relación many-to-many)
├── id (PRIMARY KEY)
├── user_id (FK → users.id)
├── device_id (FK → devices.id)
├── device_alias (nombre personalizado)
├── is_owner (primer usuario que lo reclama)
├── claimed_at
└── UNIQUE(user_id, device_id)

└── MQTT.js (cliente MQTT browser)


FLUJO DE TRABAJO
================

1. SETUP
   └── cd backend && npm install && npm run init-db

2. DESARROLLO
   ├── Terminal 1: cd backend && npm run dev
   └── Terminal 2: cd frontend && python -m http.server 8080

3. TESTING
   ├── Abrir http://localhost:8080
   ├── Registrarse
   ├── Añadir dispositivo (MAC: AA:BB:CC:DD:EE:FF)
   └── Controlar dispositivo

4. PRODUCCIÓN
   ├── Cambiar JWT_SECRET en .env
   ├── Configurar dominio y HTTPS
   ├── Deploy backend (Heroku, Railway, DigitalOcean)
   └── Deploy frontend (Netlify, Vercel)


SEGURIDAD
=========

✓ Passwords hasheados con bcrypt (10 rounds)
✓ JWT con secret + expiración (7 días)
✓ Middleware de autenticación en rutas protegidas
✓ Validación de acceso a dispositivos
✓ Queries SQL parametrizadas (previene injection)
✓ Rate limiting en auth endpoints (10 req/15min)
✓ CORS configurado
✓ Helmet.js para headers de seguridad
✓ Validación de entrada en todos los endpoints
